Fun with derivatives part 1: defining the concept of a derivative in Yacas

Introduction

On this page you can use the code developed for the accompanying article, which defines a function dd that can take the derivative of a function. The function dd takes as first argument a variable, and as second argument an expression to determine the derivative of. Clicking on dd(x,Sin(x)) should readily give you the answer Cos(x) in the Yacas calculation center on the right half of this page. You can also type in expressions yourself in the calculation center. For that you have to click on that part of the page so the calculation center starts receiving your key input.

Examples

Here are a few more simple examples.

  • dd(x,x)
  • dd(x,a)
  • dd(x,a*x)
  • dd(x,a*x^2)
  • dd(x,Sin(x)+x)

The beauty of derivatives is that you can make the expressions arbitrarily complex, and still take its derivative, although this might start to take up a lot of time from the computer (but probably still less time than if you did it yourself by hand).

  • dd(x,a*x^n)
  • dd(x,Sin(a*x))
  • dd(x,1-a*x^Sqrt(2))
  • dd(x,Exp(1+Sin(a*x)))

You can verify that the above examples return the same result as the function D built in to Yacas. For example

  • dd(x,Exp(1+Sin(a*x)))
and
  • D(x) Exp(1+Sin(a*x))
should give the same results.

Extending the program

You can extend the code yourself to handle other functions not covered yet, by going to this page. An interesting example is the derivative of Sqrt(x) (the square root of x), which can be defined as: 

dd(_x,Sqrt(_y)) <-- dd(x,y)/Sqrt(y);
but also as 
dd(_x,Sqrt(_y)) <-- (1/2)*y^(1/2-1)*dd(x,y);
If you were to try these out with dd(x,Sqrt(x));, and then repeatedly calling dd(x,%), you would see the size of the resulting expressions grow much faster when using the first transformation rule than when using the second one.
