Fun with derivatives part 1: defining the concept of a derivative in Yacas

Yacas offers the possibility of defining transformation rules. Taking a derivative of a function is a process that can be easily explained through such transformation rules (although not a necessary constraint, one can define the same operation easily in other languages also). You need surprisingly few lines of code to actually define the operation of taking a derivative. We will develop the code that can be used in Yacas in this article.

Yacas of course already has similar functionality. One can take a derivative through the D function. But it is still a good example for showing how to use transformation rules in Yacas, as very few lines of code are needed and most people readily know the transformation rules by heart.

We will define the function dd, which will take two arguments. The first is the variable, the second the function to take the derivative off (with respect to the variable given as first argument).

First we can tell the system that taking the derivative of a variable with respect to itself (eg. taking the derivative of x with respect to x) is one: 

dd(_x,_x) <-- 1;
The pattern on the left side has two matchable places named _x. This instructs Yacas to check that they are the same.

Next we tell the system that if you take a derivative of a function with respect to a variable that that function does not depend on, the result is zero. Taking the derivative of say f(y) with respect to x would yield the result zero, assuming that f(y) does not depend on x. 

dd(_x,y_IsFreeOf(x)) <-- 0;
Note the construct y_IsFreeOf(x). Yacas internally transforms that construct to mean IsFreeOf(x,y). So this pattern matches only if the expression to take the derivative off does not depend on the variable to take the derivative with respect of. The result is zero in that case.

Taking derivatives is a linear operation. The derivative of an expression that adds two expressions is the derivative of the one plus the derivative of the other. This is summarized in the following rules: 

dd(_x,_a + _b)  <-- dd(x,a)+dd(x,b);
dd(_x,_a - _b)  <-- dd(x,a)-dd(x,b);
dd(_x,   - _b)  <--        -dd(x,b);

Taking the derivative of a function to the power of a constant is also a well-known transform. Taking the derivative of x^n with respect to x yields n*x^(n-1). This can be generalized to taking the derivative of f(x)^n by applying the chain rule: d(x,f(g(x))) is d(x,g)*d(g,f), symbolically. We will embed the chain rule within the transformation rules: 

dd(_x,_y ^ z_IsFreeOf(x)) <-- dd(x,y)*z*y^(z-1);

Taking the derivative of an expression that consists of one expression multiplied by the other also has a commonly known transformation rule: 

dd(_x,_a * _b)  <-- a*dd(x,b)+dd(x,a)*b;
And so does division: 
dd(x,_y / _z) <-- (z*dd(x,y)-y*dd(x,z))/z^2;

Finally we add the derivatives of a few well-known analytic functions: 

dd(_x,Sin(_y)) <--  dd(x,y)*Cos(y);
dd(_x,Cos(_y)) <-- -dd(x,y)*Sin(y);
dd(_x,Exp(_y)) <--  dd(x,y)*Exp(y);
This list can be expanded further of course.
